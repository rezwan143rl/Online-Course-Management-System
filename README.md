# Online Course Management System

## Project Overview

This project is about developing an Online Course Management System which is a web based platform for helping both students and instructors to manage course related activities in one place. In many universities and institutions, different tasks like assignment submission, grading and announcements are done in different platforms which sometimes creates confusion and inefficiency.

So the main idea of this project is to bring everything together into a single system where academic activities can be handled in more organized way. Instructors can manage courses, upload assignments and evaluate students work, while students can enroll in courses, submit assignments and check their grades and feedback.

---

## Objectives

* To design a centralized system for managing academic courses
* To make communication between students and instructors more easier
* To reduce complexity of using multiple platforms for course activities
* To apply software engineering and object oriented design concepts in real project
* To build a scalable system which can handle many users and courses

---

## Users of the System

###  Students

Students will use the system to take part in different academic activities. They can:

* Register account and login securely
* Browse available courses and enroll into them
* Submit assignments before given deadline
* View grades and feedback given by instructors
* Check announcements related to their courses

### Instructors

Instructors will use the system to manage their courses and monitor student progress. They can:

* Create and manage courses
* Upload assignments for students
* Review submissions and give grades
* Post important announcements
* Keep record of student performance

---

## Main Features

* User registration and login system
* Course creation and enrollment functionality
* Assignment upload and submission system
* Grading and feedback management
* Course announcements feature
* Viewing course details and updates

---

##  Non Functional Requirements

The system should have simple and user friendly interface so that users can easily understand how to use it. Security is also important because personal information of users must be protected. The system should be able to respond fast even when multiple users are using it at same time. Also the architecture should support scalability so more courses and users can be added in future without major changes.

---


## Future Improvements

In future, more features can be added like better dashboard for users, performance analytics for students and possible mobile application integration. This project can be extended further to make it more practical and industry level.
