================================================================
  QUESTIONS SIR WILL ASK - SHORT ANSWERS
================================================================

Q: Who is the Subject in Observer?
A: CourseNotifier. DatabaseNotifier and AttendanceNotifier
   are the Observers.

Q: Why not put access checks directly in dashboard.php?
A: Every place accessing course content would need the same
   checks copied. If threshold changes, one line in Proxy
   changes. Nothing else.

Q: Why Decorator not Strategy for grading?
A: Strategy only allows one algorithm at a time. We need
   multiple policies active together. Decorator stacks.

Q: Where is Factory actually used?
A: At login. DB row has a role string. UserFactory::create()
   reads it, switch block returns correct subclass.

Q: Why Builder not just a constructor?
A: Constructor with all parameters is unreadable. Builder
   separates each step. Director decides which steps run.

Q: Prove Singleton.
A: Open test_singleton.php. Same object ID three times.
   === comparison true.

Q: What is the difference between Proxy and Facade?
A: Facade simplifies a complex interface. Proxy controls
   access to an existing interface. Both wrap something
   but for different reasons.

Q: How does Composite help here?
A: Without it, display code needs: if section, loop it;
   if item, show it. With Composite, both implement the same
   interface. render() on a section cascades to all children
   automatically.
